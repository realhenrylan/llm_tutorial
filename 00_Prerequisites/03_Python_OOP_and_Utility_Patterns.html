📦 什么是类（Class）？

通俗理解：类就像一个模具，用来批量生产具有相同特征的对象。比如用同一个模具做饼干。

在LLM中的作用：用来封装配置、状态、结果等，让代码更整洁、可复用。

🎨 可视化演示：类就像做饼干

形状：圆形
大小：中等

🍪 饼干1 (实例1)
形状：圆形
大小：中等
🍪 饼干2 (实例2)

Class是模板，实例是根据模板创建的具体对象。修改模板会影响所有实例，但每个实例可以有自己的独特属性！

📝 代码示例

# 定义一个类（模具）
class BatchSummary:
    def __init__(self, loss, step, extras):
        self.loss = loss
        self.step = step
        self.extras = extras
    
    def short(self):
        return f"step={self.step}, loss={self.loss:.4f}"

# 创建实例（用模具做饼干）
summary = BatchSummary(loss=0.1234, step=12, extras={"lr": 1e-4})
print(summary.short())  # 输出: "step=12, loss=0.1234"

📝 Dataclass是什么？

通俗理解：Dataclass就像一个自动帮你写好基础代码的智能模具，不用手动写__init__、__repr__等方法。

在LLM中的作用：用来快速定义配置类、数据类，减少样板代码。

🎨 可视化演示：普通类 vs Dataclass

📝
普通类
需要手动写
__init__、__repr__等
⚡
Dataclass
自动生成
__init__、__repr__等

Dataclass是Python 3.7+的特性，适合定义只存储数据的类！

📝 代码示例

# 使用dataclass装饰器
from dataclasses import dataclass

# 定义dataclass
@dataclass
class RunConfig:
    lr: float = 1e-3
    batch_size: int = 32
    epochs: int = 3

    def as_dict(self):
        return {
            'lr': self.lr,
            'batch_size': self.batch_size,
            'epochs': self.epochs,
        }

# 使用dataclass
config = RunConfig()
print("默认配置：", config.as_dict())
print("repr：", config)

🔧 工具模式是什么？

通俗理解：工具模式就像工具箱，把常用的小功能整理在一起，需要的时候随时拿出来用。

在LLM中的作用：用来组织辅助函数、工具函数，让代码更模块化。

🎨 可视化演示：工具模式

🧰 工具箱 (Utility)
format_run_tag(name, epochs)
count_tokens(tokens)
merge_config(base, overrides)
⬇️
format_run_tag
生成运行标签
count_tokens
统计token数量
merge_config
合并配置字典

工具函数是无状态的，每次调用都只依赖输入参数，不会保存中间状态！

📝 代码示例

# 工具函数示例

def format_run_tag(name, epochs):
    return f'{name}@{epochs}'

def merge_model_config(base_config, overrides):
    merged = base_config.copy()
    merged.update(overrides)
    return merged

def count_token_frequency(tokens):
    freq = {}
    for token in tokens:
        freq[token] = freq.get(token, 0) + 1
    return freq

# 使用工具函数
print(format_run_tag('TinyLM', 3))
print(merge_model_config({'name': 'TinyLM'}, {'lr': 1e-4}))

❓ 什么时候该封装成类？

通俗理解：当数据和操作它们的方法需要一起行动时，就该封装成类。就像把工具箱和说明书放在一起。

在LLM中的作用：判断何时用函数、何时用类，写出更清晰的代码。

🎨 可视化演示：函数 vs 类的决策

🔧
用函数
  • • 简单的一次性操作
  • • 无状态计算
  • • 通用工具
⚖️
决策点
状态需要保存吗？
多个方法共享数据？
📦
用类
  • • 需要保存状态
  • • 多个方法共享数据
  • • 需要统一接口

如果函数开始反复传同一组状态，就说明该封装成类了！

📝 代码示例：从函数到类

# 方式1：使用函数（简单场景）
def format_run_tag(name, epochs):
    return f'{name}@{epochs}'

# 方式2：使用类（需要更多功能）
from dataclasses import dataclass

@dataclass
class RunBuilder:
    name: str
    epochs: int

    def summary(self):
        return f'{self.name}@{self.epochs}|steps={self.epochs}'

# 使用
tag = format_run_tag('TinyLM', 3)
builder = RunBuilder(name='TinyLM', epochs=3)
print('函数：', tag)
print('类：', builder.summary())
